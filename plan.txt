# План реализации проекта cpp-reference-benchmark

## 1. Создание репозитория на GitHub
- Завести новый публичный репозиторий с названием `cpp-reference-benchmark`.
- Добавить краткое описание: "Performance benchmarks and examples demonstrating C++ references and move semantics".
- Инициализировать с README.md и лицензией MIT/Apache 2.0.
- Склонировать репозиторий локально.

## 2. Настройка структуры проекта
Создать каталоги и файлы согласно архитектуре:

## 3. Написание файлов конфигурации
- **`.gitignore`**: исключить временные файлы сборки, IDE, исполняемые файлы.
- **Корневой `CMakeLists.txt`**:
  - Установить стандарт C++20.
  - Включить опции для сборки бенчмарков, тестов и примеров.
  - В Debug-режиме добавить флаги AddressSanitizer и UndefinedBehaviorSanitizer.
  - Подключить подкаталоги `src`, `tests`, `benchmarks`, `examples`.

## 4. Реализация основной библиотеки
### 4.1. Заголовочные файлы (`include/cpp-reference-benchmark/`)
- **`buffer.hpp`**: объявления функций:
  - `sum_by_value` (принимает вектор по значению).
  - `sum_by_const_ref` (принимает const ссылку).
  - `sum_by_rvalue` (принимает rvalue-ссылку).
  - `element_at` (возвращает ссылку на элемент вектора с проверкой границ).
- **`json_utils.hpp`**: объявления функций:
  - `get_field_copy` (возвращает `std::optional<std::string>`).
  - `get_field_view` (возвращает `std::optional<std::string_view>`).

### 4.2. Исходные файлы (`src/`)
- Реализовать объявленные функции.
- В `sum_by_*` использовать `std::accumulate`.
- В `element_at` бросать исключение при выходе за границы.
- В функциях парсинга JSON реализовать простейший поиск подстрок для имитации извлечения поля.

### 4.3. `src/CMakeLists.txt`
- Создать статическую библиотеку `cpp-reference-benchmark_lib`.
- Указать публичную директорию include.

## 5. Написание модульных тестов
### 5.1. Настройка GoogleTest
- В `tests/CMakeLists.txt` подключить GoogleTest через FetchContent.
- Создать исполняемый файл `run_tests`.

### 5.2. Тестовые сценарии
- **`test_buffer.cpp`**:
  - Проверка корректности суммирования через const ref.
  - Проверка, что `element_at` возвращает ссылку, позволяющую модифицировать элемент.
  - Проверка выброса исключения при неверном индексе.
- **`test_json_utils.cpp`**:
  - Проверка извлечения значения через `get_field_copy`.
  - Проверка извлечения значения через `get_field_view`.

## 6. Реализация бенчмарков
### 6.1. Настройка Google Benchmark
- В `benchmarks/CMakeLists.txt` подключить Google Benchmark через FetchContent.
- Создать исполняемый файл `cpp-reference-benchmark_benchmarks`.

### 6.2. Бенчмарк-кейсы
- **`buffer_benchmark.cpp`**:
  - `BM_SumByValue`: измерение передачи вектора по значению для размеров 2^10 и 2^16.
  - `BM_SumByConstRef`: измерение передачи по const ссылке.
  - `BM_SumByRvalue`: измерение с явным созданием временного объекта.
  - Установка `state.SetItemsProcessed` для учёта объёма данных.
- **`json_parser_benchmark.cpp`**:
  - `BM_GetFieldCopy`: многократный вызов копирующей версии.
  - `BM_GetFieldView`: вызов версии с string_view.

## 7. Пример использования (`examples/demo.cpp`)
- Показать базовое применение функций библиотеки.
- Вывести результаты в консоль.

## 8. Настройка непрерывной интеграции (GitHub Actions) (ОПЦИОНАЛЬНО)
### 8.1. Файл `.github/workflows/ci.yml`
- Триггеры на push и pull request в ветку main.
- **Job: build-and-test**:
  - Матрица компиляторов: g++ и clang++.
  - Установка зависимостей (cmake, ninja).
  - Конфигурация CMake в Debug-режиме.
  - Сборка проекта.
  - Запуск тестов через ctest.
  - Сухой прогон бенчмарков с минимальным временем.
- **Job: static-analysis** (опционально):
  - Установка clang-tidy.
  - Генерация `compile_commands.json`.
  - Запуск clang-tidy на всём проекте.

## 9. Оформление документации (`README.md`)
- **Цель проекта**: краткое описание.
- **Теоретическая часть**: таблица сравнения типов ссылок, их назначение.
- **Структура репозитория**: дерево каталогов с пояснениями.
- **Инструкция по сборке**:
  - Требования: CMake 3.16+, компилятор C++20.
  - Команды конфигурации и сборки.
  - Запуск тестов и бенчмарков.
- **Результаты бенчмарков**: вставить текстовый вывод или скриншоты с анализом.
- **Выводы**: как правильное использование ссылок влияет на производительность.

## 10. Финальные шаги
- Проверить компиляцию без ошибок и предупреждений.
- Убедиться, что все тесты проходят.
- Запустить бенчмарки локально, сохранить результаты в файл `benchmark_results.txt` и добавить в репозиторий.
- Закоммитить все изменения и запушить в удалённый репозиторий.
- Убедиться, что GitHub Actions отрабатывает успешно.
